# 農場灌溉系統三輸出 SOP

每當用戶提交新農場評估表（農場灌溉評估.html）後，按以下順序產出三個檔案。

## 輸入來源

讀取 `農場灌溉評估.html` 的 JS state 物件，取得：
- `region`, `soilType`, `slope`
- `farmArea`, `treeArea`, `treeSpacing`, `maxDistance`
- `tankSize`, `tankCount`, `wellMotorHP`, `numMotors`, `irrigationMotorHP`
- `mainPipeSize`
- `crops` (作物種類)
- 有機/無機、施肥方式

## 系統設計推算（在生成前先算好）

| 參數 | 計算方式 |
|------|----------|
| 棵數 | treeArea×10000 / (間距²) |
| 滴箭數 | 棵數 × 4（四爪滴箭，預設每棵4個） |
| 灌溉分區數 | ceil(maxDistance / 30) 或依流量決定 |
| 控制點數 | 1"閥數 + 3"閥數 + 灌溉馬達數 |
| 主幹管長度 | maxDistance × 1.2（加10%裕量） |
| PE支管長度 | 棵數 × 平均株距 × 0.8 |

## 輸出1：規劃圖 PNG

**腳本位置：** `{案場資料夾}/output/gen_diagram.py`

**圖形結構（單路滴灌）：**
```
地下水 → 地下水馬達（既有）→ 儲水桶（既有）→ 灌溉馬達 → 主幹管 → 1"閥×N → 滴灌支管 → 滴箭×M
```

**字體規則（不可違反）：**
- box() fs1≥18, fs2≥14
- tank() fontsize≥14
- motor_circ() fontsize≥13
- 標題列 fontsize≥21
- 閥號碼 fontsize≥7
- 每次生成後必須用 Read 工具預覽圖片確認可讀性

**輸出路徑：** `{案場資料夾}/output/{案場名稱}_規劃圖.png`

## 輸出2：報價單 HTML

**模板基礎：** `C:\Users\cowle\OneDrive\Desktop\農場灌溉報價.html`（梓官農場版本）

**必含功能：**
- 互動式BOM表（可啟用/停用項目）
- 折扣切換
- 補助計算（水利署IA + 農糧署AFA）
- 控制點費用（每點NT$4,500）
- 免責聲明：補助金額依地方不同

**不含：** 灌溉建議/時程（屬於思維檢查的功能，不在報價單中）

**佔位符（缺資料時使用）：**
- `[農場名稱]`、`[作物種類]`、`[地址]`

**輸出路徑：** `{案場資料夾}/output/{案場名稱}_報價單.html`

## 輸出3：思維檢查流程 HTML

**內容結構（5區塊）：**
1. 水源確認 — 水量、水質、備源
2. 輸送系統 — 管路壓損、馬達揚程、流速檢驗
3. 灌溉需求 — 作物需水、土壤AWC、灌溉頻率
4. 電控系統 — 控制點數、自動化等級
5. 維護保養 — 過濾頻率、季節性停機

**格式：** 可勾選清單，每項目含「通過/警告/不適用」三態，JS驅動，無後端

**輸出路徑：** `{案場資料夾}/output/{案場名稱}_思維檢查.html`

---

**Why:** 每個農場案場需要相同三文件，標準化流程避免遺漏關鍵設計驗證步驟
**How to apply:** 用戶提交新農場評估表後，不需詢問直接按此SOP產生三個輸出；使用`[佔位符]`處理缺失資料
