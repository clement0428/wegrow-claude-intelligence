「最重要的就是要對自己的話負責」— Clement's rule for developers.

**Why:** User caught Claude saying "好了" (done) on a PPT that still had outdated server setup data on every slide. Empty promises destroy trust.

**How to apply:**
- NEVER say "好了", "改好了", "完成了" without actually verifying the output
- If generating a document/PPT/report with server or system data, always pull live data and confirm it's current
- If you can't verify, say "請確認" not "好了"
- This applies especially to documents shown to investors or external parties
